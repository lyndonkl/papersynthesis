## Headline
Benchmarks three retrieval architectures — Vector RAG, Tree Reasoning, and a proposed LLM-routed Adaptive Hybrid Retrieval (AHR) — across financial, legal, and medical documents, and argues that query complexity (not domain or paradigm) determines which retrieval strategy wins, with no single paradigm dominating every tier.

## Summary
The paper formalizes a four-tier query taxonomy (T1 single-fact → T4 multi-document synthesis) and reports that Tree Reasoning achieves the highest aggregate LLM-as-judge quality on a controlled 22-query corpus (0.900) and on a stratified n=50 sample from FinanceBench (0.938), while AHR — an LLM-based router that dispatches by tier to vector / tree / fused retrieval — places second on both corpora (0.845 and 0.901). The author argues that prior contradictory rankings in the literature (Lumer et al. 2025: vector wins 68%; PageIndex 2025: tree wins 98.7%) are an artifact of different tier coverage rather than a real disagreement, and that AHR's value is "catastrophic-failure avoidance" — being a Pareto-improvement on latency (3.10s vs 3.40s for Tree Reasoning) and lacking the low-score outliers each pure paradigm exhibits — rather than aggregate-score dominance. Findings are presented as directionally consistent between the controlled corpus and FinanceBench, with the Tree-vs-Vector quality gap widening from 5.5 to 11.7 percentage points on real SEC filings.

## Specifics
- **Architectures compared.** Vector RAG (FAISS + `all-MiniLM-L6-v2`, top-5 cosine retrieval); Tree Reasoning RAG (derived from PageIndex / VectifyAI — documents pre-parsed into a hierarchical tree of LLM-generated node summaries, navigated at query time by an LLM with regex-based cross-reference following); Adaptive Hybrid Retrieval (AHR) — a GPT-4o-mini query classifier assigns each query a tier and routes T1→vector, T2-T3→tree, T4→fused. Both generator and judge are GPT-4o-mini.
- **Tier-level results (controlled corpus, n=22).** Tree Reasoning leads T1 (0.938); AHR leads T2 (0.929) and T3 (0.850); Vector RAG leads T4 (0.900). The author frames this as the central evidence that "no single paradigm dominates all query complexity tiers."
- **Aggregate results (controlled corpus).** Tree 0.900 quality / 0.977 recall / 0.850 precision / 3.40s latency; Vector 0.845 / 0.932 / 0.341 / 1.61s; AHR 0.845 / 0.955 / 0.655 / 3.10s. Vector's precision (0.341) is far below its recall (0.932) — the system retrieves the right section but pulls in noise; the LLM-as-judge appears robust to this, which the paper does not discuss.
- **FinanceBench validation (n=50, real SEC 10-K/10-Q).** Tree 0.938 / AHR 0.901 / Vector 0.821 on LLM-as-judge quality. The author reports the controlled ranking is "confirmed and strengthened" on real filings, with AHR routing shifting from 55%/45% tree/vector on the controlled corpus to 74%/26% on FinanceBench — the author notes this "reflects the greater structural complexity of real SEC filings" but does not address whether the classifier was calibrated on the controlled corpus (a potential leakage path).
- **Structural-capability claim.** Tree-based methods reach 100% cross-reference recall on T3 queries vs 91.7% for Vector RAG; the author argues vector similarity cannot replicate cross-reference following in principle because cross-referenced sections have low semantic similarity to the query. With n=22 total controlled queries, only a subset are T3, so the 91.7% figure is one or two queries from 100% — directionally fine, fragile in magnitude.
- **Numerical reasoning is the universal weakness.** On FinanceBench's "logical reasoning (numerical)" subset, all three architectures perform comparably poorly (Tree 0.533, Vector 0.500, AHR 0.500) — the paper frames multi-step numerical reasoning as the unsolved category.
- **Tension between title and headline result.** The title and contribution list foreground AHR as the proposed system, but AHR loses to Tree Reasoning on every aggregate metric (0.845 vs 0.900 controlled; 0.901 vs 0.938 FinanceBench). The defense rests on tier-specific wins (T2, T3), latency improvement, and the catastrophic-failure-avoidance argument (anecdotal: Vector RAG scoring 0.20 on some medical queries, Tree Reasoning scoring 0.60 on some multi-document queries).
- **Clarity / reproducibility flags.** Figures 1–5 in the rendered PDF show the same composite three-panel image while each caption describes a different analysis (tier breakdown / domain breakdown / latency-quality / cross-reference / Domain×Tier heatmap). The Domain×Tier heatmap claimed in Figure 5 is not visible. The tier-level numbers anchoring the paper's central thesis sit in prose and Tables 2–5 only; the figures don't render them. No statistical significance testing is reported; the author flags scaling to the full 150-question FinanceBench set with significance testing as Future Work.
- **Methodological caveats worth carrying forward.** (a) LLM-as-judge and answer-generator share a model family (GPT-4o-mini), and the router is also GPT-4o-mini — judge/generator alignment bias is not discussed; (b) on FinanceBench, the paper reports Vector RAG at 0.798 accuracy on its own 0–1 LLM-as-judge scale but does not normalize against prior FinanceBench results (FinSage 49.66% pass/fail accuracy), so the "directionally consistent" comparison to prior SOTA is unfalsifiable without scale harmonization; (c) eight references are 2026-dated Medium / Substack / VectifyAI blog posts — the peer-reviewed evidentiary base is thinner than the 52-reference list suggests.
- **Why this matters for the agentic-retrieval lane.** The AHR router is itself an LLM-driven retrieval primitive — query classification by an LLM (GPT-4o-mini) before dispatching to vector or tree backends — which puts the paper inside the "agentic RAG" pattern even though the paper does not use that vocabulary. The structural-capability framing (cross-reference following as something dense retrieval cannot do in principle) is a claim about the limits of pure vector retrieval that the lane should track.
