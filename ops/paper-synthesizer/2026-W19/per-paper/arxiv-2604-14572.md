## Headline
Corpus2Skill compiles an enterprise document corpus offline into a filesystem-shaped hierarchy of LLM-written skill summaries that an agent then top-down navigates at serve time, and reports highest F1 against dense, hybrid, agentic-RAG and RAPTOR baselines on WixQA — with the win explicitly scoped to atomic, single-domain, 0.5–3k-character document corpora rather than claimed as universal.

## Summary
The argument is that for enterprise QA over single-domain corpora the right primitive is *navigation*, not retrieval: the corpus is compiled once into a skill forest (`SKILL.md` / `INDEX.md` / `documents.json` files built via iterative bottom-up K-Means clustering plus LLM summarization at each level), and an agent then top-down browses it via two tools (`code_execution` for file viewing, `get_document` for full-text lookup), with progressive disclosure keeping per-turn token cost bounded while the agent retains a corpus-wide map. The evidence is a 200-query head-to-head on WixQA (6,221-doc Wix customer-support corpus) where Corpus2Skill posts F1=0.468 / Factuality=0.739 / Context Recall=0.673 — leading all six metrics over Hybrid, Dense, Agentic, RAPTOR and BM25 — followed by a 9-subset RAGBench generalization sweep that *qualifies* the headline by exposing corpus shapes where flat retrieval still wins (homogeneous tables, long extractive contracts, open-domain pools). The authors are consistent in characterizing a regime rather than claiming universal superiority — "complementary design point," "right primitive for…," and TatQA / CUAD are reported as honest losses rather than buried.

## Specifics
- **Named system and serve-time surface.** Corpus2Skill = a compile-then-navigate pipeline. Compile phase: embed docs → ⌈n/p⌉ K-Means partitioning at branching ratio p → LLM-summarize each cluster → re-embed summaries → repeat until ≤K top-level clusters remain → materialize as files. Serve phase: an LLM agent with two tools (`code_execution`, `get_document`) reads the tree top-down. The positioning row the authors lean on (Table 8) is "LLM only" serve infrastructure — no vector DB, no cluster index, no graph DB at query time.
- **Headline numbers (WixQA, n=200).** Corpus2Skill F1 0.468 vs Agentic 0.381 (≈23% lift), RAPTOR 0.389, Dense 0.363, Hybrid 0.360, BM25 0.342. Factuality 0.739 vs Agentic 0.719. Context Recall 0.673 vs RAPTOR 0.616. Cost/query $0.089 vs Agentic $0.088 — but cost parity is *only* with Anthropic ephemeral prompt caching enabled; without caching $/query is $0.172, roughly 2× Agentic. The authors disclose this in §5.
- **The regime claim (DC5), defended in both directions.** Navigation beats flat retrieval when the corpus is (i) single-domain, (ii) topically coherent at the document level, (iii) atomic (one feature/entity/how-to per doc), and (iv) 0.5–3k characters. Elsewhere — TatQA homogeneous tables (Corpus2Skill F1 0.246 vs Dense 0.304 at ~80× the cost, $0.969 vs $0.005, because the agent hits max_turns on indistinguishable financial-statement clusters), CUAD long extractive contracts, CovidQA/ExpertQA open-domain — the authors explicitly recommend flat retrieval or RAPTOR.
- **Macro-average is narrow, not a sweep (Table 3).** Across 10 datasets, Corpus2Skill leads on every lexical-overlap metric (F1 0.350, BLEU 0.103, R-1 0.378, R-2 0.177) but the F1 lead over Hybrid is 0.001. Agentic wins macro Factuality 0.660 (the authors *suggest* the judge rewards "I don't know" fallbacks Agentic produces); Dense wins Context Recall 0.540 (the authors *suggest* the judge rewards span overlap that embedding retrieval directly optimizes). Both are framed as hypotheses, not measurements.
- **Ablations support DC4 — "quality lives in the compiled tree, not in the navigator."** Branching ratio p=5/10/20 stays within ~0.02 F1; max-rounds budget 5/10/20 stays within ~0.01 F1; swapping Sonnet 4.6 for Haiku 4.5 retains 91% F1 at 71% cost (Context Recall actually rises with Haiku, 0.698 vs 0.673). The authors interpret this as the hierarchy carrying the signal, with the navigator being relatively interchangeable.
- **Dominant failure mode is top-level routing error.** 38 of 62 WixQA failures stem from the agent choosing the wrong top-level branch, not from within-branch search. Hard single-path clustering (each doc assigned to exactly one branch) is called out as a known limitation with cross-cutting docs; soft / multi-parent clustering is floated as future work.
- **Compilation cost is reported but thin.** ~6.5 min wall-clock on 32-CPU + $5–$10 in LLM calls for WixQA; no breakdown of clustering vs summarization vs labeling, no token accounting, no scaling characterization. For an "invest at compile time" thesis this is a load-bearing number that is underspecified.
- **Skill-API constraints shape the design space.** Anthropic Skills API limits (8 skills/request, 200 files/skill, 30 MB/skill) directly motivate K=7 top-level skills and "compact mode" — parent `INDEX.md` inlines near-leaf children rather than creating subdirectories. The framework's design space is partly captured by one vendor's product limits; the paper says so in §5.
- **Hedging-vs-title gap.** The body consistently characterizes a regime ("complementary design point," "right primitive for…"). The title "Don't Retrieve, Navigate" is a stronger universal imperative than the body defends. This is the one place the paper promotes beyond what the evidence carries.
- **Open questions noted by the extractor.** No confidence intervals or seeds reported on the 200-query samples; judge-artifact explanations for Factuality and Context Recall losses are asserted but not measured; WixQA labels co-developed with Wix product taxonomy may flatter the clusterer; cost-parity vs Agentic is caching-conditional.
- **Lineage and adjacent cluster.** Methodologically descends from RAPTOR's iterative embed-cluster-summarize loop, but materializes the hierarchy as files an agent navigates rather than as a vector-indexed tree. Sits inside a 2025-26 navigation-RAG cluster the extractor flags as worth triangulating (NaviRAG, A-RAG, SPD-RAG, HCAG, BookRAG, HiRAG, StructRAG), and adopts the (C, π, T, R) agent-skill formalism from Jiang et al. 2026, reinterpreting it from procedural to informational.
