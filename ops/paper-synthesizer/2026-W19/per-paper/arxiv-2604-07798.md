# Pass A — Within-paper layered summary

## 30,000 ft (paper's claim, one sentence)
Provides a small-language-model-driven memory system for LLM agents with three-tier (short/mid/long-term) storage, two-stage retrieval, and online/offline separation under bounded compute.

## 3,000 ft (argument + evidence, 2-3 sentences)
LightMem positions itself between two failure modes: pure retrieval-based memory (low overhead, unstable accuracy) and large-model memory operations (accurate, slow). Its design modularizes memory retrieval, writing, and consolidation; separates online processing from offline consolidation; uses three-tier memory (STM/MTM/LTM); supports multi-user via identifier-keyed retrieval. Online retrieval uses a vector-coarse + semantic-consistency-rerank two-stage procedure under a fixed budget. LightMem reports an average +2.5 F1 improvement over A-MEM on LoCoMo at 83 ms retrieval / 581 ms end-to-end latency.
