## Headline

LACONIC pushes learned-sparse retrieval back into the conversation: an 8B Llama-3-based LSR model hits 60.2 nDCG on MTEB Retrieval (#15 on the leaderboard at time of submission) using 71% less index memory than an equivalent dense model, on commodity CPUs.

## Summary

Learned sparse retrieval has been the underdog to dense retrieval for years. LACONIC closes the gap with a two-phase training curriculum on top of Llama-3 backbones (1B/3B/8B sizes): (1) weakly supervised pre-finetuning to adapt causal LLMs for bidirectional contextualization, then (2) high-signal finetuning with curated hard negatives. The result is a sparse retriever that ranks #15 on the MTEB Retrieval leaderboard while using a fraction of the memory of equivalent dense models, and crucially runs on CPU.

For a practitioner: if your retrieval cost is dominated by GPU inference for dense retrievers or RAM for the vector index, LACONIC is the most concrete invitation in the kept set to revisit learned-sparse retrieval. The 71% memory savings and CPU-friendly inverted-index serving are practical, not just academic.

## Specifics

- Model family: LACONIC 1B / 3B / 8B (Llama-3-based, learned sparse retrievers).
- Training: two-phase curriculum — weakly-supervised pre-finetuning, then hard-negative finetuning.
- 8B variant: 60.2 nDCG on MTEB Retrieval, #15 on the leaderboard at submission (Jan 1, 2026).
- Memory: 71% less index memory vs equivalent dense model.
- Hardware: CPU-deployable (no GPU required for inference).
