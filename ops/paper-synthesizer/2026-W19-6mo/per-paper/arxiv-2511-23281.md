## Headline

In a controlled e-commerce testbed, RAG, MCP, and NLWeb agents all outperform raw HTML browsing on both accuracy and efficiency — there is no single winner among the three structured interfaces, but HTML loses cleanly.

## Summary

A four-way controlled comparison that did not exist before: simulated e-shops expose products via HTML, MCP, and NLWeb interfaces; for each interface the authors built a specialized agent and ran identical task sets across GPT-4.1, GPT-5, GPT-5-mini, and Claude Sonnet 4. Tasks range from simple product search to complex substitute/complementary queries and full checkouts. The winner depends on task and model, but RAG, MCP, and NLWeb all outperform HTML on both effectiveness and efficiency.

For a practitioner: if you are building an agent that interacts with e-commerce sites, raw HTML browsing should be the fallback path, not the default. MCP and NLWeb show the field's real direction — exposing structured interfaces specifically for agents — and RAG over pre-crawled content remains a strong baseline when the site does not expose structured access.

## Specifics

- Four interfaces: HTML, RAG (over pre-crawled content), MCP (Model Context Protocol API), NLWeb (natural-language querying).
- Four LLM backbones: GPT-4.1, GPT-5, GPT-5 mini, Claude Sonnet 4.
- Testbed: four simulated e-shops; tasks include product search, price comparison, complementary/substitute product queries, full checkout.
- Headline: RAG, MCP, NLWeb all beat HTML on both accuracy and efficiency.
- Method note: the same agent code runs against four interface flavors per shop — clean controlled-experiment design.
