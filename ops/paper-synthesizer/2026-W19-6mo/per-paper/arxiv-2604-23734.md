## Headline

Prism-Reranker emits more than a score: when a document is judged relevant, it also produces a contribution statement and a self-contained evidence-passage rewrite — eliminating the need to dump entire documents into the LLM context.

## Summary

Modern retrieval pipelines feed RAG and agentic systems that need more than a scalar score — they need to know how a document helps and which passage carries the signal. A reranker that only emits "how relevant" forces the agent to dump entire documents into context, wasting tokens. Prism-Reranker (Qwen3.5 backbone, 0.8B/2B/4B/9B sizes) emits three things when its verdict is yes: the standard relevance judgement, a contribution statement summarizing how the document helps, and an evidence passage — a self-contained rewrite that preserves query-relevant signal while dropping noise. Trained with hybrid distillation (point-wise from a commercial reranker API) plus SFT on contribution and evidence targets. Reports +1.54 NDCG@10 average on BEIR-QA when used on Qwen3-Reranker-4B.

For a practitioner: this changes what the reranker output is, not just how good it is. If your agent currently passes full retrieved documents into the LLM context, Prism-Reranker's evidence-passage rewrite eliminates that token cost in one step. The four-size model release also means you can match the reranker capacity to your latency budget.

## Specifics

- Output: relevance verdict + contribution statement + evidence passage rewrite.
- Backbone: Qwen3.5; sizes 0.8B / 2B / 4B / 9B.
- Training: hybrid — point-wise distillation from commercial reranker + SFT on contribution/evidence targets.
- Headline: +1.54 average NDCG@10 on BEIR-QA when applied to Qwen3-Reranker-4B.
- Eliminates token waste from passing full documents into context.
