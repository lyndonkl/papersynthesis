## Headline

A-RAG exposes three retrieval tools — keyword search, semantic search, and chunk read — as parallel agent-callable primitives, letting the agent orchestrate retrieval at multiple granularities and scale with model capability.

## Summary

Existing RAG falls into two patterns: single-shot retrieval that concatenates passages into the input, or a predefined workflow the model executes step by step. Neither lets the model decide retrieval-time. A-RAG exposes three retrieval primitives — keyword search (for exact-match), semantic search (for conceptual match), chunk read (for local context) — as agent-callable tools. The agent adaptively chooses among them across multiple granularities. Experiments on multiple open-domain QA benchmarks show A-RAG matches or exceeds existing approaches with comparable or lower retrieved tokens.

For a practitioner: A-RAG is the cleanest expression of the "give the agent the tools, let it orchestrate" pattern in the kept set. If you are using a frontier model whose tool-calling is reliable, exposing these three primitives separately is more flexible than picking one retriever upfront. The paper also shows A-RAG scales with model size and test-time compute, which is the property practitioners want for an agent system that gets better as the underlying model improves.
