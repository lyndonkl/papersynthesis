## Headline

Higress-RAG is the practitioner-targeted reference architecture for 2026 — a Model Context Protocol-based stack assembling Adaptive Routing, Hybrid Retrieval (dense+sparse via RRF), Semantic Caching at 50ms, and Corrective RAG into one production pipeline.

## Summary

This paper is not a single-technique contribution — it is the integration play. Built on the Model Context Protocol, the Higress RAG MCP Server stitches together five independent advances into one stack: Adaptive Routing (decide which retrieval path), Hybrid Retrieval (dense + sparse merged via Reciprocal Rank Fusion), Higress-Native Splitter (structure-aware chunking), Semantic Caching (50ms latency target), and Corrective RAG (CRAG) for hallucination resistance. The contribution is the assembly, not any one piece.

For a practitioner: if you are designing a production-grade RAG system in 2026 and want a complete reference architecture rather than a single technique, this is the paper to read. - Corrective RAG (CRAG): integrated for hallucination resistance.
