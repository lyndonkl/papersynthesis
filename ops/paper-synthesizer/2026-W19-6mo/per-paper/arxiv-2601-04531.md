## Headline

Self-MedRAG codifies a transferable recipe — BM25 + Contriever + Reciprocal Rank Fusion + NLI-based self-reflection — for iterative hybrid retrieval; useful as a template even if you are not in medical QA.

## Summary

The medical domain is the venue, but the architecture is general. Self-MedRAG combines sparse (BM25) and dense (Contriever) retrievers via Reciprocal Rank Fusion, generates an answer with supporting rationales, and lets a lightweight self-reflection module (NLI or LLM verification) decide whether the rationale is supported. If not, the system reformulates the query and iterates. The benchmarks are MedQA and PubMedQA.

For a practitioner: even if you are building outside medical QA, the architecture is a clean reference for the iterative-hybrid pattern: BM25 + Contriever + RRF for retrieval breadth, NLI for evidence verification, query reformulation for self-correction. Easy to lift into other domains.

## Specifics

- Hybrid retrieval: BM25 (sparse) + Contriever (dense) via Reciprocal Rank Fusion.
- Self-reflection: lightweight module using NLI or LLM-based verification of rationale support.
- Iteration: query reformulation if rationale lacks support.
- Benchmarks: MedQA, PubMedQA.
- Pattern is portable to non-medical domains.
