# Per-Run Synthesis Style Override: 2026-W20

This override **replaces** the standing `shared-context/synthesis-style.md` for this run only. Self-contained: the synthesizer reads only this file.

This file carries the standing 30K / 3K / 300 ft layered-reasoning rules **plus** a fourth required block per cluster — the **audience-reframe translation** that bridges the classical systems-thinking finding into the operator's working vocabulary (hierarchical multi-agent orchestration: MapReduce-style fan-out / fan-back-in / spread-out). The reframe is the substantive deliverable of this run.

---

## Layered reasoning structure (unchanged from standing rules)

The reader of the digest has three different questions on different days. The three layers serve those three questions, in this order:

| Layer       | The reader's question                            | The unit of writing      |
| ----------- | ------------------------------------------------ | ------------------------ |
| 30,000 ft   | "What did you find across all the papers?"       | The run                  |
| 3,000 ft    | "What were the themes? Tell me by topic."        | The cluster              |
| 300 ft      | "Show me the actual papers."                     | The individual paper     |

The synthesis is **top-down**. Write 30K first, then 3K per cluster, then 300 ft per paper. Lower layers must be consistent with the upper layer.

### Layer 1 — 30,000 ft: What did you find across all the papers?

One paragraph (3–5 sentences). Answers: **what is the through-line of this run, and how does it connect classical systems thinking to the reader's multi-agent orchestrators?**

Required elements:

1. The through-line across the kept papers — one sentence. If no through-line, say so.
2. What did NOT change / what the field is still stuck on / what's still missing.
3. What's continuing from prior digests, if anything.
4. Where the disagreement lies, if any.

**This run's anti-patterns to reject:**

- Listing cluster names with one sentence each. (That's 3K.)
- Naming individual papers by author. (That's 300 ft.)
- Hedging with "Various developments occurred" / "Several papers explored."
- Adjectives without arguments: "important," "exciting," "novel."
- **Banned vocabulary** (do not use): *delve, unpack, paradigm shift, let's explore, moreover, furthermore, it's worth noting.*

### Layer 2 — 3,000 ft: What were the themes?

For each cluster from `paper-cluster-by-theme`: 2–3 sentences answering **what does this cluster of papers collectively argue?**

Required elements per cluster:

1. State the cluster's collective claim.
2. Cite the strongest 1–2 papers inline.
3. Surface tension if any (where papers disagree on magnitude / mechanism).
4. Tag continuity if the cluster appeared in any of the last 4 digests: `[continuing from 2026-Wxx]`.

Then — **NEW for this run** — the audience-reframe block (mandatory; see next section).

### Layer 3 — 300 ft: Show me the actual papers

For each paper in each cluster, one bullet:

```
- **{Title}** — {first author et al.}, {source}, {YYYY-MM-DD}. {one sentence on what it does
  and why it belongs in this cluster.} → [link]({url})
```

The one-sentence description must:

1. Say what the paper *does* (method? result? benchmark? review?), not what it's "about."
2. Connect to the cluster's argument — why this paper landed here.
3. Preserve the paper's own hedging. "Suggests" stays "suggests"; never upgrade to "shows."

## NEW: Per-cluster audience-reframe block (the substantive addition for this run)

After each cluster's 3K paragraph (and before that cluster's 300 ft paper bullets), insert a **`#### For your orchestrator`** sub-block answering the operator's four working questions. This is the run's translation deliverable — the bridge from classical systems-thinking finding to operational guidance for hierarchical multi-agent orchestration (MapReduce-style fan-out / fan-back-in / repeated spread).

The block is short and pointed (≤ 200 words, often shorter). Four labeled lines, in this order:

```
#### For your orchestrator

- **Classical concept it maps to:** {Name the canonical systems-thinking primitive the
  cluster's finding operationalizes. Examples: balancing loop, reinforcing loop, leverage
  point, requisite variety, Goodhart drift, stock/flow imbalance, modular decomposition,
  cascading failure, critical slowing, weak-link / strong-link, edge of chaos. Name it,
  then in one clause say why this cluster is the modern empirical/theoretical face of it.}

- **What to observe:** {The signal in your orchestrator that this cluster's finding tells
  you to instrument. Specific: which artifact, which trace property, which inter-agent
  message pattern, which queue depth, which retry rate. Not "monitor the system."}

- **What to tune to improve it:** {The lever this cluster identifies. Specific: branching
  factor at a fan-out level, message coupling between lanes, barrier placement, lane
  count, retry policy, escalation threshold. Tie to the classical concept's leverage
  point.}

- **Failure mode this catches early — and the feedback loop:** {Name the failure mode this
  cluster characterizes in classical terms (cascade, oscillation, lock-in, Goodhart,
  thrashing, premature convergence, herd, deadlock). State the early-warning signal AND
  the automated correction loop: which metric, what threshold, which action — closing
  the loop. Not "log it."}
```

**Constraints:**

- Every bullet must be *operational*. If a bullet could be rewritten as "monitor X" or "watch out for Y" without losing meaning, it's too vague — rewrite. The reader wants handles, not advice.
- Name the **classical concept** explicitly. The reader recently read *Thinking in Systems* and wants the vocabulary reinforced, not abstracted away.
- If a cluster genuinely doesn't translate (e.g., a methodological paper on percolation thresholds that has no actionable orchestrator analog), say so honestly in one sentence and skip the four-bullet block: `> No clean orchestrator translation — this cluster is field-internal methodology. Read at the 3K level for context only.`

**Anti-patterns (reject):**

- Vague advice ("monitor coordination," "watch for instability") — name the specific signal.
- Bullets that just restate the cluster's 3K paragraph in different words.
- Bullets that reach for a fashionable buzzword instead of the classical concept (e.g., "antifragile" used loosely where the cluster is actually about **redundancy with diversity**).
- LLM-specific framings ("prompt engineer your agents to..."). This is systems-theoretic guidance; the orchestrator the reader is building happens to use LLMs but the reframe is layer-agnostic.

## Cross-layer consistency checks (run before writing the file)

1. **Upward:** does the 30K paragraph reflect what the clusters collectively say?
2. **Downward:** does each cluster's 3K paragraph match the papers in its 300 ft list?
3. **Lateral:** do two cluster names contradict? Surface tension in 30K if so.
4. **Reframe consistency:** does each cluster's audience-reframe block name a *different* classical concept than its neighbors? If two clusters reframe to the same primitive (e.g., both invoke "balancing loop"), one of two things: either the clusters should merge, or the reframe is mis-aimed. Decide which and act.

If any check fails, fix before writing.

## Run-level closing section (after all clusters)

Following the last cluster, write a closing **`## Closing: How this run lands for your orchestrator`** section — 4–8 sentences. The reader needs to leave with a *prioritized handful*, not a checklist.

Required elements:

1. **The two or three highest-leverage instrumentations** they should add to their orchestrator now, drawn from the run's clusters (cite the cluster names).
2. **The one failure mode they're most likely to hit first** given that they build MapReduce-style hierarchical orchestrators — pulled from the run's evidence, named in classical terms, with the cluster citation.
3. **The feedback loop that wraps the above** — the single automated loop that, if they only build one, gives them the most early-warning coverage.
4. **What this run did NOT cover** (honest gap list, since the operator wants fundamentals → today and arXiv/PubMed don't carry the full classical canon).

---

## Banned vocabulary (do not use, anywhere in the digest)

delve, unpack, paradigm shift, let's explore, moreover, furthermore, it's worth noting, in conclusion, importantly, fundamentally (when used as an adjective rather than a real claim).
