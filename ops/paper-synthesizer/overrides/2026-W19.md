# Topic for this run

Latest retrieval techniques and re-ranking techniques **for LLM agents**, published in the **last 6 months** (2025-11-10 → 2026-05-10), with focus on work that is **highly cited or gaining traction**.

The operator wants the digest framed as **today's theme** — what stands out from the 6-month window as the cohort of work that matters.

## What "for LLM agents" means in scope

In scope (the agentic angle is required, not optional):
- Retrieval-augmented generation (RAG) where the retrieval surface is being designed for or driven by an LLM agent (agent memory, tool-use retrieval, multi-step / iterative retrieval, agentic search).
- Re-ranking layers (cross-encoders, neural rerankers, listwise rerankers) where the optimization target is downstream LLM utility, agent decision quality, or end-task accuracy — not classical IR relevance only.
- Retrieval pipeline components (dense passage retrieval, hybrid sparse-dense, query understanding/expansion, long-context retrieval) when evaluated in an agent setting or proposed as an agent primitive.
- Agent memory retrieval: how an agent stores, indexes, and recalls past interactions / observations / tool outputs.
- Retrieval interface design (graph-based, hierarchical, skill-tree, no-embedder/grep-the-corpus, MCP-style tool access) for agents.
- Efficiency of agentic retrieval (latency, search depth, failure repair, orchestration cost) when it speaks to retrieval/reranking architecture.

Out of scope:
- Pure information retrieval theory or classical search-engine work without an LLM/agent angle.
- Pre-training of dense embedders absent retrieval evaluation in a downstream LLM setting.
- Workshop summaries, position pieces, or commentary without empirical or methodological contribution.

## Traction signal

The relevance filter and the digest are oriented around two tiers:

- **Established** — papers that have visibly accumulated citations, follow-on work, or direct adoption in subsequent papers within the 6-month window. (Older papers in the window have an inherent advantage here; that's expected and reflects "established.")
- **Emerging** — recent papers (last ~6-8 weeks) where the citation count cannot yet be high, but the paper either (a) is from a recognized group, (b) proposes a clear novel mechanism, (c) reports a sizable headline number with named benchmark, or (d) is being referenced by other recent papers in the kept set.

The orchestrator enriches each KEEP with a Semantic Scholar citation count before Mode B reads the per-paper summaries. The digest's clustering and prioritization should reflect this two-tier framing — see `2026-W19-style.md` for the rendering rules.
